vue基础知识点总结(一)

发表于 2020-08-01 更新于 2020-08-06 分类于 Vue Valine：
本文字数： 13k 阅读时长 ≈ 12 分钟

创建一个vue实例，Vue是一个构造函数

Vue指令

v-cloak

解决网络不好情况下用户看到插值表达式的情况，解决闪烁的问题

v-text

默认v-text是没有闪烁问题的，但是会覆盖原有的内容，插值表达式不会覆盖内容，但有闪烁问题，这是这两者的区别，两者也都不能解析html标签

v-bind

Vue提供用于绑定属性的指令

v-on

Vue提供了 v-on 事件绑定机制

事件修饰符

.stop 阻止冒泡
.prevent 阻止默认事件
.self 只当在 event.target 是当前元素自身时触发处理函数，即事件不是从内部元素触发的
.once 点击事件将只会触发一次
.passive 滚动事件的默认行为 (即滚动行为) 将会立即触发，而不会等待 onScroll 完成
.stop 和 .self的区别
.stop 阻止了所有的冒泡,.self只负责自己的冒泡

v-model 双向数据绑定

v-bind 只能实现数据的单向数据绑定,从m自动绑定到v，无法实现数据的双向数据绑定
使用v-model 指令，可以实现表单元素和model中数据的双向数据绑定
注意v-model 只能运用在表单元素中
input(radio,text,address,email…) select checkbox textarea

v-for 循环

在使用v-for循环数组或对象的时候，记得绑定 :key 唯一标识，数组用 of 分隔符，对象用 in 分隔符

v-if 和 v-show

v-if 的特点是每次都会删除或创建元素
v-show 的特点每次不会重新进行dom的删除和创建，只是切换了display的样式
v-if 有较高的切换性能消耗
v-show 有较高的初始渲染消耗
如果元素涉及到频繁的切换不要使用 v-if ，推荐使用 v-show
如果元素可能永远不会被显示出来被看到，推荐使用 v-if

Vue 计算属性、监听、方法

1、computed:计算属性是有缓存机制的，如果涉及到的值没有改变，不会重新计算，页面不会重新渲染，可以监听多个变量。

2、watch:监听也是有缓存机制的，每次只能监听单个属性的变化，代码过于麻烦，如果三种都可以实现，建议使用计算属性。

watch 监听复杂数据类型 object array 深度监视

3、methods:方法没有缓存机制，每次都要重新渲染页面。

4、如果涉及到vue实例外的数据发生变化，computed正常是不会重新渲染页面，但是如果watch监听里的变量如果改变，会导致computed重新计算渲染，单独修改arr值是不会发生变化，当watch监听到的值发生变化是，computed会重新计算。

组件传值

父传子 props

父组件，可以在引用组件的时候，通过属性绑定的形式，把需要传递给子组件的数据以属性绑定的形式，传递到子组件内部，供子组件使用。

子传父 $emit()

$emit(方法名，参数(可选))，向上一层触发，在子组件上用 v-on 绑定 emit() 传过来的方法名，父组件用另一个方法名去接收方法和参数。
